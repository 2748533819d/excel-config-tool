# Excel Config Tool - 提取模式详解

> 本文档详细说明每种提取模式的配置方式、使用场景和注意事项

---

## 模式总览

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           提取模式总览                                           │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  模式              |  输入示例      |  输出类型        |  典型场景               │
│  ─────────────────────────────────────────────────────────────────────────────  │
│  SINGLE            |  [A1]         |  Object          |  标题、汇总值            │
│  DOWN              |  [A2,A3,A4]   |  Array<Object>   |  列数据提取              │
│  RIGHT             |  [A1,B1,C1]   |  Array<Object>   |  表头、横向数据          │
│  BLOCK             |  [A1:C10]     |  Array<Array>    |  数据矩阵、交叉表        │
│  UNTIL_EMPTY       |  [A2,A3...∅]  |  Array<Object>   |  动态长度列表            │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

---

## 1. SINGLE - 单一单元格模式

### 1.1 配置结构

```yaml
cells:
  title:
    key: "title"
    position:
      cellRef: "A1"
    mode: SINGLE
    parser:
      type: string
    required: false
```

### 1.2 使用场景

```
┌─────────────────────────────────────────┐
│  订单汇总表                              │
│  ┌─────────────────────────────────┐    │
│  │  订单汇总表        ← A1 (title)  │    │
│  │  2024 年 1 月       ← A2 (month)  │    │
│  │  合计：¥100,000    ← A3 (total)  │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

### 1.3 完整配置示例

```yaml
version: "1.0"
templateName: "订单汇总表"
cells:
  # 提取标题
  title:
    key: "title"
    position:
      cellRef: "A1"
    mode: SINGLE
    parser:
      type: string
    defaultValue: ""
    
  # 提取月份
  month:
    key: "month"
    position:
      cellRef: "A2"
    mode: SINGLE
    parser:
      type: date
      params:
        pattern: "yyyy 年 M 月"
        
  # 提取总计金额
  totalAmount:
    key: "totalAmount"
    position:
      cellRef: "A3"
    mode: SINGLE
    parser:
      type: number
      params:
        prefix: "合计：¥"  # 去除前缀后解析数字
```

### 1.4 提取结果

```json
{
  "title": "订单汇总表",
  "month": "2024-01-01T00:00:00Z",
  "totalAmount": 100000
}
```

### 1.5 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 单元格为空 | 返回 `null` 或 `defaultValue` |
| 类型不匹配 | 解析失败，返回错误信息 |
| 单元格不存在 | 返回 `null` |

### 1.6 前端交互

```
用户操作：点击 A1 单元格
        ↓
组件显示：[A1] 被选中（高亮）
        ↓
配置面板自动填充:
  - 位置：A1
  - 模式：SINGLE (自动识别)
  - 字段名：[用户输入]
```

## 2. DOWN - 向下提取模式

### 2.1 配置结构

```yaml
cells:
  orderNos:
    key: "orderNos"
    position:
      cellRef: "A2"
    mode: DOWN
    range:
      skipEmpty: true  # 遇到空行停止（推荐）
      maxRows: 1000    # 可选：最多读取 1000 行（安全限制）
    parser:
      type: string
```

**关键说明**：
- `rows` 参数**不是必须的**，配置时通常不知道具体有多少行数据
- 默认使用 `skipEmpty: true`，一直读取直到遇到空行
- `maxRows` 是可选的安全限制，防止读取过多数据

### 2.2 使用场景

```
┌─────────────────────────────────────────┐
│  订单明细表                              │
│  ┌───────┬────────┬────────┬─────────┐  │
│  │ 订单号 │ 金额   │ 日期   │ 客户    │  │
│  ├───────┼────────┼────────┼─────────┤  │
│  │ ORD001│ 100.00 │01-01   │ 张三    │ ← 第 1 行数据
│  │ ORD002│ 200.00 │01-02   │ 李四    │ ← 第 2 行数据
│  │ ORD003│ 300.00 │01-03   │ 王五    │ ← 第 3 行数据
│  │ ...   │ ...    │ ...    │ ...     │
│  │ ORD100│ ...    │ ...    │ ...     │ ← 第 100 行数据
│  └───────┴────────┴────────┴─────────┘  │
│     ↑                                     │
│     从 A2 开始向下提取 100 行                   │
└─────────────────────────────────────────┘
```

### 2.3 完整配置示例

```yaml
version: "1.0"
templateName: "订单明细表"
cells:
  # 订单号列 - 使用 skipEmpty 自动检测行数
  orderNos:
    key: "orderNos"
    position:
      cellRef: "A2"
    mode: DOWN
    range:
      skipEmpty: true  # 推荐：遇到空行自动停止
      maxRows: 1000    # 安全限制：最多 1000 行
    parser:
      type: string
    handlers:
      - type: trim
      - type: notEmpty
        params:
          message: "订单号不能为空"
          
  # 金额列 - 同样的方式，行数由数据决定
  amounts:
    key: "amounts"
    position:
      cellRef: "B2"
    mode: DOWN
    range:
      skipEmpty: true
      maxRows: 1000
    parser:
      type: number
      params:
        scale: 2
        
  # 日期列
  dates:
    key: "dates"
    position:
      cellRef: "C2"
    mode: DOWN
    range:
      skipEmpty: true
      maxRows: 1000
    parser:
      type: date
      params:
        pattern: "MM-dd"
        
  # 客户列
  customers:
    key: "customers"
    position:
      cellRef: "D2"
    mode: DOWN
    range:
      skipEmpty: true
      maxRows: 1000
    parser:
      type: string
```

**说明**：所有列都使用 `skipEmpty: true`，实际提取的行数由数据决定，而不是配置写死。
如果某个文件有 50 行数据，就提取 50 行；有 500 行就提取 500 行（不超过 maxRows）。

### 2.4 提取结果

```json
{
  "orderNos": ["ORD001", "ORD002", "ORD003", ...],
  "amounts": [100.00, 200.00, 300.00, ...],
  "dates": ["2024-01-01", "2024-01-02", "2024-01-03", ...],
  "customers": ["张三", "李四", "王五", ...]
}
```

### 2.5 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 指定行数内遇到空值 | `skipEmpty: true` 时跳过，`false` 时停止 |
| 实际行数不足 | 返回实际提取的行数 |
| 某列中间有空值 | 该位置返回 `null` 或跳过 |

### 2.6 前端交互

```
用户操作：选中 A2:A100 区域
        ↓
组件自动识别:
  - 起始位置：A2
  - 方向：向下
  - 行数：99 (自动计算)
        ↓
配置面板建议:
  - 模式：DOWN
  - 行数：[99] (可修改)
  - 跳过空值：[✓]
```

## 3. RIGHT - 向右提取模式

### 3.1 配置结构

```yaml
cells:
  monthHeaders:
    key: "monthHeaders"
    position:
      cellRef: "A1"
    mode: RIGHT
    range:
      skipEmpty: true  # 遇到空列停止（推荐）
      maxCols: 100     # 可选：最多读取 100 列（安全限制）
    parser:
      type: string
```

**关键说明**：
- `cols` 参数**不是必须的**，配置时通常不知道具体有多少列数据
- 默认使用 `skipEmpty: true`，一直读取直到遇到空列
- `maxCols` 是可选的安全限制

### 3.2 使用场景

```
┌─────────────────────────────────────────────────────────────┐
│  月度销售报表                                                │
│  ┌──────┬─────┬─────┬─────┬─────┬─────┬ ... ┬─────┐        │
│  │ 月份  │ 1 月 │ 2 月 │ 3 月 │ 4 月 │ 5 月 │ ... │ 12 月 │        │
│  ├──────┼─────┼─────┼─────┼─────┼─────┼     │─────┤        │
│  │ 销售额│ ... │ ... │ ... │ ... │ ... │ ... │ ... │        │
│  └──────┴─────┴─────┴─────┴─────┴─────┴     │─────┘        │
│         ↑                                   ↑               │
│         从 A1 开始向右提取 12 列 (1-12 月的表头)                    │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 完整配置示例

```yaml
version: "1.0"
templateName: "月度销售报表"
cells:
  # 月份表头 (1-12 月)
  monthHeaders:
    key: "monthHeaders"
    position:
      cellRef: "B1"
    mode: RIGHT
    range:
      cols: 12
    parser:
      type: string
      
  # 销售额数据
  salesAmounts:
    key: "salesAmounts"
    position:
      cellRef: "B2"
    mode: RIGHT
    range:
      cols: 12
    parser:
      type: number
      params:
        scale: 2
        
  # 目标额数据
  targetAmounts:
    key: "targetAmounts"
    position:
      cellRef: "B3"
    mode: RIGHT
    range:
      cols: 12
    parser:
      type: number
      params:
        scale: 2
```

### 3.4 提取结果

```json
{
  "monthHeaders": ["1 月", "2 月", "3 月", "4 月", "5 月", "6 月", "7 月", "8 月", "9 月", "10 月", "11 月", "12 月"],
  "salesAmounts": [1000, 1200, 1500, 1300, 1600, 1800, 2000, 1900, 2100, 2300, 2500, 2700],
  "targetAmounts": [1000, 1000, 1200, 1200, 1500, 1500, 1800, 1800, 2000, 2000, 2200, 2200]
}
```

### 3.5 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 指定列数内遇到空值 | 返回 `null` (不跳过) |
| 实际列数不足 | 返回实际提取的列数 |
| 超出 Excel 最大列数 (XFD) | 抛出错误 |

### 3.6 前端交互

```
用户操作：选中 B1:M1 区域 (1-12 月表头)
        ↓
组件自动识别:
  - 起始位置：B1
  - 方向：向右
  - 列数：12 (自动计算)
        ↓
配置面板建议:
  - 模式：RIGHT
  - 列数：[12] (可修改)
  - 字段名：[monthHeaders]
```

## 4. BLOCK - 区域块模式

### 4.1 配置结构

```yaml
cells:
  dataMatrix:
    key: "dataMatrix"
    position:
      areaRef: "B2:M13"
    mode: BLOCK
    parser:
      type: number
```

### 4.2 使用场景

```
┌─────────────────────────────────────────────────────────────────┐
│  产品销售矩阵                                                    │
│  ┌────────┬──────┬──────┬──────┬──────┬──────┬─────────────────┤
│  │ 产品\月 │ 1 月  │ 2 月  │ 3 月  │ 4 月  │ ...  │ 12 月            │
│  ├────────┼──────┼──────┼──────┼──────┼──────┼                 │
│  │ 产品 A  │ 100  │ 120  │ 150  │ 130  │ ...  │  ← 第 1 行数据    │
│  │ 产品 B  │ 200  │ 220  │ 250  │ 230  │ ...  │  ← 第 2 行数据    │
│  │ 产品 C  │ 300  │ 320  │ 350  │ 330  │ ...  │  ← 第 3 行数据    │
│  │ ...     │ ...  │ ...  │ ...  │ ...  │ ...  │                │
│  │ 产品 M  │ ...  │ ...  │ ...  │ ...  │ ...  │  ← 第 12 行数据   │
│  └────────┴──────┴──────┴──────┴──────┴──────┴                 │
│           ↑                              ↑                       │
│           区域块 B2:M13 (12 行 x 12 列 数据矩阵)                       │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 完整配置示例

```yaml
version: "1.0"
templateName: "产品销售矩阵"
cells:
  # 提取整个数据矩阵 (12x12 二维数组)
  salesMatrix:
    key: "salesMatrix"
    position:
      areaRef: "B2:M13"
    mode: BLOCK
    parser:
      type: number
      params:
        scale: 2
        
  # 只提取第一行 (产品 A 的月度数据)
  productAData:
    key: "productAData"
    position:
      areaRef: "B2:M2"
    mode: BLOCK
    parser:
      type: number
```

### 4.4 提取结果

```json
{
  "salesMatrix": [
    [100, 120, 150, 130, 160, 180, 200, 190, 210, 230, 250, 270],
    [200, 220, 250, 230, 260, 280, 300, 290, 310, 330, 350, 370],
    [300, 320, 350, 330, 360, 380, 400, 390, 410, 430, 450, 470],
    ...
  ],
  "productAData": [[100, 120, 150, 130, 160, 180, 200, 190, 210, 230, 250, 270]]
}
```

### 4.5 数据结构说明

```
BLOCK 模式输出 = 二维数组

[                    // 整个矩阵
  [值，值，值，...],  // 第 1 行
  [值，值，值，...],  // 第 2 行
  [值，值，值，...],  // 第 3 行
  ...
]

访问方式:
matrix[0][0]  = 第 1 行第 1 列
matrix[2][5]  = 第 3 行第 6 列
```

### 4.6 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 区域内有空单元格 | 该位置返回 `null` |
| 区域超出实际范围 | 超出的部分返回 `null` |
| 单行区域 (如 A1:M1) | 返回 `[[值，值，值...]]` (仍为二维) |

### 4.7 前端交互

```
用户操作：拖拽选中 B2:M13 区域
        ↓
组件自动识别:
  - 区域引用：B2:M13
  - 行数：12
  - 列数：12
        ↓
配置面板建议:
  - 模式：BLOCK
  - 区域：[B2:M13] (自动填充)
  - 数据类型：[number]
```

## 5. UNTIL_EMPTY - 直到空值模式

### 5.1 配置结构

```yaml
cells:
  dynamicList:
    key: "dynamicList"
    position:
      cellRef: "A2"
    mode: UNTIL_EMPTY
    range:
      skipEmpty: false  # false=遇到空值停止，true=跳过空值继续
    parser:
      type: string
```

### 5.2 使用场景

```
┌─────────────────────────────────────────┐
│  不定长数据列表                            │
│  ┌─────────────────────────────────┐    │
│  │ 产品名称 (不定长)                │    │
│  ├─────────────────────────────────┤    │
│  │ 产品 A       ← A2 (开始)        │    │
│  │ 产品 B       ← A3               │    │
│  │ 产品 C       ← A4               │    │
│  │ ...         (数量不定)           │    │
│  │ 产品 X       ← A100             │    │
│  │            ← A101 (空，停止)     │    │
│  │            ← A102               │    │
│  └─────────────────────────────────┘    │
│                                         │
│  不确定有多少行数据，直到遇到空行为止         │
└─────────────────────────────────────────┘
```

### 5.3 完整配置示例

```yaml
version: "1.0"
templateName: "不定长产品列表"
cells:
  # 产品名称列表 (不定长)
  productNames:
    key: "productNames"
    position:
      cellRef: "A2"
    mode: UNTIL_EMPTY
    range:
      skipEmpty: false  # 遇到第一个空值就停止
    parser:
      type: string
    handlers:
      - type: trim
      
  # 对应的价格列表 (同样不定长)
  prices:
    key: "prices"
    position:
      cellRef: "B2"
    mode: UNTIL_EMPTY
    range:
      skipEmpty: false
    parser:
      type: number
      params:
        scale: 2
```

### 5.4 提取结果

```json
{
  "productNames": ["产品 A", "产品 B", "产品 C", ..., "产品 X"],
  "prices": [10.00, 20.00, 30.00, ..., 100.00]
}
```

### 5.5 skipEmpty 行为对比

```
数据:
A2: "值 1"
A3: ""       (空)
A4: "值 2"
A5: "值 3"
A6: ""       (空)

skipEmpty: false (遇到空值停止)
结果：["值 1"]  ← A3 是空值，停止，A4 及之后的不读取

skipEmpty: true (跳过空值继续)
结果：["值 1", "值 2", "值 3"]  ← 跳过 A3，继续读取，直到 A6 停止
```

### 5.6 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 起始单元格就是空值 | 返回空数组 `[]` |
| 整列都是空值 | 返回空数组 `[]` |
| 最后一行之后还有数据 | 只提取到第一个空值前 |

### 5.7 前端交互

```
用户操作：选中 A2 单元格 (单个单元格)
        ↓
组件提示:
  "检测到单单元格选择，是否使用 UNTIL_EMPTY 模式？"
  "该模式将向下提取直到遇到空值"
        ↓
用户确认:
  - 模式：UNTIL_EMPTY
  - 跳过空值：[□] (默认 false)
  - 字段名：[dynamicList]
```

## 6. 模式对比总结

### 6.1 选择指南

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              模式选择指南                                        │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  你要提取什么？                                                                 │
│                                                                                 │
│  单个值 (标题、汇总数)                                                           │
│  └──→ SINGLE                                                                    │
│                                                                                 │
│  一列数据 (纵向)                                                                 │
│  ├──→ 确定行数？                                                                │
│  │   ├── 是 → DOWN (指定 rows)                                                  │
│  │   └── 否 → UNTIL_EMPTY (直到空值)                                             │
│                                                                                 │
│  一行数据 (横向，如表头)                                                          │
│  └──→ RIGHT (指定 cols)                                                         │
│                                                                                 │
│  一个区域 (矩阵、交叉表)                                                          │
│  └──→ BLOCK (指定 areaRef)                                                      │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 6.2 特性对比表

| 特性 | SINGLE | DOWN | RIGHT | BLOCK | UNTIL_EMPTY |
|------|--------|------|-------|-------|-------------|
| **输出类型** | Object | Array | Array | Array<Array> | Array |
| **需要指定范围** | 否 | 是 (rows) | 是 (cols) | 是 (areaRef) | 否 |
| **支持 skipEmpty** | N/A | ✓ | N/A | N/A | ✓ |
| **自动识别** | ✓ | ✓ | ✓ | ✓ | △ (需确认) |
| **典型场景** | 标题 | 列数据 | 表头 | 矩阵 | 不定长列表 |

### 6.3 组合使用示例

```yaml
# 实际项目中通常是多种模式组合使用
version: "1.0"
templateName: "综合报表"
cells:
  # 1. 提取标题 (SINGLE)
  reportTitle:
    key: "reportTitle"
    position:
      cellRef: "A1"
    mode: SINGLE
    parser:
      type: string
      
  # 2. 提取月份表头 (RIGHT)
  monthHeaders:
    key: "monthHeaders"
    position:
      cellRef: "B1"
    mode: RIGHT
    range:
      cols: 12
    parser:
      type: string
      
  # 3. 提取产品名称列 (DOWN)
  productNames:
    key: "productNames"
    position:
      cellRef: "A2"
    mode: DOWN
    range:
      rows: 100
      skipEmpty: true
    parser:
      type: string
      
  # 4. 提取销售数据矩阵 (BLOCK)
  salesData:
    key: "salesData"
    position:
      areaRef: "B2:M101"
    mode: BLOCK
    parser:
      type: number
      
  # 5. 提取备注列 (UNTIL_EMPTY)
  remarks:
    key: "remarks"
    position:
      cellRef: "N2"
    mode: UNTIL_EMPTY
    parser:
      type: string
```

## 7. 前端配置流程

### 7.1 智能模式识别

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                        前端智能模式识别流程                                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  用户选择           →  自动识别  →  建议模式  →  用户确认/修改                  │
│                                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────────┐   │
│  │  选择类型           |  识别逻辑           |  建议模式                     │   │
│  ├─────────────────────────────────────────────────────────────────────────┤   │
│  │  单个单元格         |  1x1 区域            |  SINGLE                       │   │
```

## 8. 注意事项

### 8.1 性能考虑

| 模式 | 性能特点 | 优化建议 |
|------|----------|----------|
| SINGLE | 最快 | 无 |
| DOWN | 中等 | 指定合理行数，不要过大 |
| RIGHT | 快 | 列数通常较小 |
| BLOCK | 较慢 | 避免过大的区域 |
| UNTIL_EMPTY | 不确定 | 大数据建议用 DOWN+ 固定行数 |

### 8.2 内存优化

```
SAX 流式读取时的内存控制:

• SINGLE/DOWN/RIGHT/UNTIL_EMPTY: 逐行处理，内存占用低
• BLOCK: 需要缓存整个区域，内存占用较高

建议:
• BLOCK 模式区域不要超过 1000 行 x 100 列
• 超大数据集使用 DOWN 模式分批提取
```
