# 葡萄城 GcExcel + SpreadJS 架构设计分析

## 一、产品定位与架构思想

### 1.1 产品矩阵

葡萄城的表格解决方案采用**前后端分离 + 协同处理**的架构：

```
┌─────────────────────────────────────────────────────────────────┐
│                    葡萄城全栈表格解决方案                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────┐         ┌─────────────────────┐       │
│   │    SpreadJS         │         │     GcExcel         │       │
│   │    纯前端表格控件    │         │   服务端表格组件     │       │
│   │                     │         │                     │       │
│   │   • HTML5/JS        │         │   • .NET / Java     │       │
│   │   • 类 Excel 界面     │         │   • 服务端处理       │       │
│   │   • 模板设计         │         │   • 批量导出        │       │
│   │   • 数据展示交互     │         │   • 复杂计算        │       │
│   │   • 双缓存画布引擎   │         │   • 不依赖 Office   │       │
│   │   • 行模式稀疏矩阵   │         │   • 高性能渲染      │       │
│   └─────────────────────┘         └─────────────────────┘       │
│              ↕                                ↕                 │
│         前端展示/交互                    后端处理/导出             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 核心设计思想

| 思想 | 说明 |
|------|------|
| **前后端同构** | 前后端使用统一的数据模型和 API 设计 |
| **任务分担** | 前端负责交互展示，后端负责批量处理 |
| **数据同步** | 前后端 JSON 数据格式完全兼容 |
| **零依赖** | 不依赖 Microsoft Office 或 POI |

### 2.4 存储引擎 - 稀疏矩阵

```javascript
// 传统二维数组 vs 稀疏矩阵存储

// ❌ 传统方式：10000x100 的表格需要 100 万元素
const traditional = new Array(10000).fill(null).map(() => new Array(100));

// ✅ SpreadJS 方式：只存储有数据的单元格
class SparseMatrix {
    constructor() {
        this._data = new Map(); // row -> Map(col -> CellData)
    }
    
    set(row, col, value) {
        if (!this._data.has(row)) {
            this._data.set(row, new Map());
        }
        this._data.get(row).set(col, value);
    }
    
    get(row, col) {
        const rowMap = this._data.get(row);
        return rowMap ? rowMap.get(col) : undefined;
    }
}

// 内存对比:
// 10000 行 x 100 列，实际只有 1000 个单元格有数据
// 传统方式：100 万 个引用
// 稀疏矩阵：1000 个 实际数据
```

### 5.2 核心设计原则借鉴

| 葡萄城原则 | 本项目应用 |
|------------|------------|
| **前后端同构** | 配置格式前后端统一 (JSON Schema) |
| **任务分担** | 前端配置 + 预览，后端提取 + 处理 |

```
│  │    • 输出：confirm (确认导出)                              │ │
│  │    • 不涉及：实际导入导出逻辑                              │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  后端组件 (Spring Boot + POI)                                    │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  ConfigParser:                                            │ │
│  │    • 职责：解析 YAML/JSON 配置                               │ │
│  │    • 不涉及：配置存储方式                                  │ │
│  │                                                           │ │
│  │  ExcelReader:                                             │ │
│  │    • 职责：SAX 流式读取 Excel                               │ │
│  │    • 不涉及：配置来源、结果用途                            │ │
│  │                                                           │ │
│  │  ExtractStrategy:                                         │ │
│  │    • 职责：执行具体提取逻辑                                │ │
│  │    • 不涉及：配置如何生成                                  │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
